Utility Billing Data Quality & Revenue Risk Analysis

Microsoft SQL Server 路 Star Schema 路 Power BI (DAX) 路 Data Quality 路 Revenue Risk

Project Overview

This project models a utility billing system in Microsoft SQL Server to analyze customer behavior, billing accuracy, revenue leakage, and compliance risks. Interactive Power BI dashboards built for analytics and operations teams.

Data Model

This project uses a star schema in Microsoft SQL Server to organize utility billing data.

  • DimCustomers
  • DimAccounts
  • DimTransactionType
  • DimDate
  • FactUsage (Fact Table)

Key Business Questions & Analyses

1. Data Inconsistencies: Flag customers marked as Residential in DimCustomer but whose accounts are Commercial or Small Commercial in DimAccount.
2. Inactive Accounts With Recent Activity: Identify accounts marked as AccountStatus = 'Inactive' but still showing usage transactions.
3. Same Customer, Multiple Accounts: Find customers who have multiple accounts (Residential + Small Commercial) under their name.
4. Refund Abuse: Identify customers where total Refund amount exceeds total Payment amount within a month. Detect cases where a Refund was issued without a corresponding Bill.
5. Critical Care Compliance: Identify Critical Care Residential customers who were ever charged a Late Fee (should ideally have no service interruptions).
