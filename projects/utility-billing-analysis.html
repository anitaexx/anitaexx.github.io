Utility Billing Data Quality & Revenue Risk Analysis

Microsoft SQL Server 路 Star Schema 路 Power BI (DAX) 路 Data Quality 路 Revenue Risk

Project Overview

This project models a utility billing system using a star schema in Microsoft SQL Server to analyze customer behavior, billing accuracy, revenue leakage, and compliance risks. Interactive Power BI dashboards surface actionable insights for analytics and operations teams.

Data Model

  • FactUsage (fact table)
  • DimCustomers
  • DimAccounts
  • DimTransactionType
  • DimDate

Key Analytical Scenarios

1. Customer & Account Data Inconsistencies

SELECT c.CustomerID, c.CustomerType, a.AccountType
FROM DimCustomers c
JOIN DimAccounts a ON c.CustomerID = a.CustomerID
WHERE c.CustomerType = 'Residential'
AND a.AccountType IN ('Commercial', 'Small Commercial');

2. Inactive Accounts With Usage

SELECT a.AccountID, a.AccountStatus, SUM(f.UsageAmount) AS TotalUsage
FROM FactUsage f
JOIN DimAccounts a ON f.AccountID = a.AccountID
WHERE a.AccountStatus = 'Inactive'
GROUP BY a.AccountID, a.AccountStatus
HAVING SUM(f.UsageAmount) > 0;

3. Refund Abuse Detection

SELECT CustomerID, MONTH(TransactionDate) AS Month,
       SUM(CASE WHEN TransactionType = 'Refund' THEN Amount ELSE 0 END) AS Refunds,
       SUM(CASE WHEN TransactionType = 'Payment' THEN Amount ELSE 0 END) AS Payments
FROM FactUsage
GROUP BY CustomerID, MONTH(TransactionDate)
HAVING SUM(CASE WHEN TransactionType = 'Refund' THEN Amount ELSE 0 END)
     > SUM(CASE WHEN TransactionType = 'Payment' THEN Amount ELSE 0 END);

4. High Outstanding Balances

SELECT CustomerID,
       SUM(Billed) - SUM(Payments) - SUM(Refunds) + SUM(LateFees) AS OutstandingBalance
FROM FactUsage
GROUP BY CustomerID
HAVING SUM(Billed) - SUM(Payments) - SUM(Refunds) + SUM(LateFees) > 1000;

5. Aging Report (30 / 60 / 90 / 90+)

CASE
  WHEN DATEDIFF(day, BillDate, GETDATE()) <= 30 THEN '0-30'
  WHEN DATEDIFF(day, BillDate, GETDATE()) <= 60 THEN '31-60'
  WHEN DATEDIFF(day, BillDate, GETDATE()) <= 90 THEN '61-90'
  ELSE '90+'
END AS AgingBucket

6. Critical Care Compliance

SELECT c.CustomerID, f.LateFeeAmount
FROM FactUsage f
JOIN DimCustomers c ON f.CustomerID = c.CustomerID
WHERE c.CriticalCareFlag = 'Yes'
AND f.LateFeeAmount
