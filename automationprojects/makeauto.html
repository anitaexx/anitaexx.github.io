AI Operations Command Center

OpenAI · Make · Airtable · Gmail · Workflow Automation

Project Overview

Built an AI-powered operations system using OpenAI that classifies issues, tracks execution in Airtable, and sends intelligent follow-ups on overdue work via Gmail. The system turns operational inputs into structured records, applies deadline logic, and supports accountability through automated reminders.

Workflow

Airtable → Make → OpenAI → JSON Parsing → Airtable Update → Gmail Follow-Up

What the System Does

  • Captures operational issues in Airtable and enriches them with AI-generated classifications and summaries.
  • Uses OpenAI to assign type, priority, owner suggestions, and recommended actions.
  • Applies logic in Make to populate deadlines and maintain consistent system fields.
  • Sends follow-up reminders on overdue work and updates reminder history for tracking.

Why I Built It

Execution often breaks when teams have no reliable way to track issues, define ownership, and close loops. I built this system to show how AI and automation can create a lightweight operations layer that improves visibility, accountability, and follow-through.

Key Outcome

This project demonstrates how AI can be used beyond content generation by powering structured operational workflows, execution tracking, and automated follow-up logic inside a real system.
